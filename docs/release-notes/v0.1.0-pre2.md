# Lyra v0.1.0-pre2 — RX2 Dual Receiver (tester pre-release)

**Tester pre-release.** This is the first build of the v0.1
line, ahead of the production v0.1.0 release.  Distributed to
Brent, Timmy, and other bench testers for real-band
confirmation before public release.

> Cumulative release history lives in [`CHANGELOG.md`](../../CHANGELOG.md).
> This file is the discrete release-note drop attached to the
> GitHub v0.1.0-pre2 release for quick reading.

## What's new

**Second receiver (RX2)** running on the HL2's DDC1 channel,
with full independence: own frequency, mode, bandwidth, AGC
profile, AF gain, volume, mute.  Listen to two frequencies at
once in stereo split (RX1 hard-left, RX2 hard-right), or have
either VFO be the "audible" one in mono mode — switch with a
single click.

### Operator-facing highlights

- **Two big VFO LEDs side by side** on the TUNING panel — RX1
  left, RX2 right, Lyra logo centered between them.  Each LED
  takes double-click + type, mouse-wheel tuning,
  click-to-focus.
- **Focus model** — green border marks the focused VFO.  Click
  the other LED, press **Ctrl+1** / **Ctrl+2**, or middle-click
  the panadapter to swap focus.  Mode picker, panadapter
  click-tune, wheel-tune, band buttons, GEN/TIME/Mem recalls,
  FT8 marker clicks, cluster spot clicks — all route to
  whichever VFO is currently focused.
- **SUB toggle** for dual-RX stereo split.  Off = mono on
  focused VFO.  On = RX1 hard-left + RX2 hard-right with
  independent **Vol RX1 / Vol RX2 / Mute-A / Mute-B** sliders
  always visible on the DSP+AUDIO panel.
- **VFO bridge buttons** (`1→2` / `2→1` / `⇄`) for copying or
  swapping VFO state.
- **CW Pitch control** moved to the TUNING panel where it
  lives more naturally next to the tuning controls.  Shared
  across both receivers; both DDCs offset correctly in CW
  modes so click-to-tune lands on the carrier on either VFO.
- **Per-VFO Step + Mode combos** under each LED — RX1 on
  band-sweep at 1 kHz while RX2 holds at 1 Hz zero-beat on a
  CW target, no problem.
- **Panadapter follows the focused RX** — including the
  passband-rectangle overlay, the orange VFO marker, the CW
  pitch offset, and click-to-tune routing.
- **Full persistence** — every RX2 state field survives Lyra
  restarts.  No more "RX2 always boots to 7.250 MHz USB"
  surprise.

### Bug fixes / quality

- **Propagation panel reliability fix.**  The HamQSL solar
  feed fetch now matches SDRLogger+'s `verify=False` posture
  for SSL cert verification, fixing the blank-panel failure
  surfaced by tester Timmy on a machine where Lyra's
  Python-stdlib urllib was rejecting hamqsl.com's cert chain
  while browsers + SDRLogger+ on the same machine accepted
  it.
- **Diagnostic output now lands in `%APPDATA%\Lyra\crash.log`**
  on the PyInstaller windowed build.  Previously every
  `[Radio] ...` print went to a `None` stderr and vanished.
  Each session writes a start-marker line for readability
  across multi-run logs.
- **Panadapter marker clicks + cluster spot activation** now
  route to the focused VFO instead of always RX1.
- **Mouse-wheel tuning + Exact / 100 Hz quantization** both
  work on the RX2 panadapter (were RX1-only).
- **GEN slot owner-RX tracking** — clicking GEN1 while
  focused on RX2 makes RX2 the slot's "owner"; tweaking RX1
  freq doesn't clobber GEN1's stored state.

### Test coverage

**225 / 225 unit tests passing** including the Phase 0
byte-identical RX1 null gate that protects against any RX1
regression from the dual-channel refactor.

## What testers should specifically try

1. **Dual-RX listening on one band.**  Tune RX1 to FT8 on 20m
   (14.074 MHz DIGU), RX2 to a CW segment (14.030 MHz CWU).
   Click SUB.  You should hear FT8 in left ear, CW in right.
2. **Cross-band dual-RX.**  RX1 on 40m phone, RX2 on 20m
   phone.  Same band roll-off applies (single HL2 front-end
   filter bank) but the digital tuning is fully independent.
3. **Focus swapping with SUB off.**  RX1 on 40m, click VFO B
   → panadapter retunes to RX2's band, audio routes to RX2,
   AF/AGC sliders now operate on RX2.  Click VFO A → snap
   back.
4. **VFO B click-to-tune in CW** specifically (this had a
   fix in late hotfixes — should land on the carrier, not
   off by the pitch tone).
5. **Persistence.**  Set RX2 to something specific, quit
   Lyra, relaunch.  RX2 should come back exactly where you
   left it.
6. **Propagation panel** (Timmy specifically) — should show
   solar numbers + band-conditions heatmap on launch.  If it
   stays blank, hover the panel for a diagnostic tooltip and
   check `%APPDATA%\Lyra\crash.log` for the actual exception.

## Known caveats

- **SPLIT / TX-on-VFO-B is not implemented yet.**  TX itself
  is v0.2 work.  The current UI cluster (`SUB` / `1→2` /
  `2→1` / `⇄`) may grow a tri-state SUB/SPLIT/OFF mode when
  TX lands — operator UX discussion ongoing, tester feedback
  welcomed.
- **Audio / panadapter / waterfall sync** may have minor
  latency skew between the three surfaces.  Investigation
  parked pending repro evidence from bench testing — please
  report anything that feels obviously out of sync.
- **HL2 front-end filter bank is shared** between RX1 and
  RX2.  Cross-band dual-RX works but expect signal-strength
  roll-off on whichever VFO is outside the active filter
  band.
- **PureSignal interaction** (v0.3 work): when PS lands,
  DDC1 is gateware-rerouted to the PA coupler during MOX+PS.
  Lyra will pause RX2 reception during MOX+PS automatically.

## Installation

1. Run `Lyra-Setup-0.1.0-pre2.exe`.
2. Choose per-user (no admin) or system-wide (admin).
3. Launch from the Start menu.
4. First launch: open ⚙ **Settings → Radio**, enter your HL2's
   IP (or click **Discover**), close Settings, click **▶ Start**.
5. Press **F1** anywhere in the app to open the in-app user
   guide.  The new **"RX2 (Dual Receiver)"** topic walks
   through the dual-receiver workflow.
