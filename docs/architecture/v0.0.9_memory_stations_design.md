# v0.0.9 — Memory & Stations design

**Status:** design doc, operator-approved scope, no code yet.
**Date:** 2026-05-02.
**Ships before:** v0.1 (RX2 work).
**Approver:** N8SDR (Rick Langford).

Pre-RX2 polish release covering four operator-requested features
that share a common thread: **stored frequency presets the
operator can quickly recall, plus auto-discovered station
information overlaid on the panadapter.**

---

## 1. Scope

Four features:

1. **Time-station cycle button** — quick-tune to WWV / CHU / BPM /
   etc., country-aware ordering, click cycles through frequencies.
2. **GEN1 / GEN2 / GEN3 customizable buttons** — right-click to
   save current freq + mode under that GEN slot; click to recall.
3. **User memory presets (20 slots)** — operator-named freq + mode
   bank, accessible from a new **Presets** button on the Bands
   panel with a dropdown selector.
4. **SW broadcaster overlay (EiBi)** — downloadable schedule
   database, panadapter labels for active broadcasters in the
   visible range, time-of-day filtering.

All four land on the **Bands panel** UI surface and persist via
QSettings.

### What this release is NOT

- Not multi-receiver — RX2 is the v0.1 release.
- Not TX-related — TX is v0.2.
- Not a full memory-bank system with banks, sub-banks, etc.  20
  flat presets covers the operating use case.
- Not band-aware auto-load (e.g., "load 40m presets when tuning
  to 40m") — operator picks explicitly.
- Not AOKI database support — EiBi only for v0.0.9; AOKI may come
  in a follow-up release if operators ask.

### 2.2 Country-based ordering algorithm

```python
def order_stations(operator_country: str) -> list[TimeStation]:
    """Return time stations in operator-priority order.

    Priority: same country -> same continent -> rest by HF
    propagation 'reachability' from typical mid-latitudes.
    """
    same_country = []
    same_continent = []
    rest = []
    op_continent = COUNTRY_TO_CONTINENT.get(
        operator_country.upper(), "")
    for s in TIME_STATIONS:
        if s.country_code == operator_country.upper():
            same_country.append(s)
        elif s.continent == op_continent:
            same_continent.append(s)
        else:
            rest.append(s)
    return same_country + same_continent + rest
```

`COUNTRY_TO_CONTINENT` is a small dict bundled in
`lyra/data/iso_country.py` (or reused if Lyra already has one).

### 2.3 Cycle button UX

New button on the Bands panel, slotted in the **"Other" row**
alongside any existing non-amateur-band buttons.  Label: **`TIME ▾`**
(with the dropdown caret hint).

**Click behavior:**

- **Plain left-click**: cycle to next frequency.  Internal index
  walks: station[0].freqs[0] → station[0].freqs[1] → ... →
  station[0].freqs[end] → station[1].freqs[0] → ... → wrap.
- **Right-click**: popup menu listing all stations grouped by
  country with their full frequency lists; click any to jump
  directly.  Also exposes:
  - "Reset to first station" (resets cycle index to 0)
  - "Hide stations I never hear..." → opens Settings → Bands →
    Time Stations sub-tab

**Tooltip during cycle:** "Tuned to WWV 10 MHz (Fort Collins, CO)"
flashed on the status bar for 2 seconds.

### 2.4 Mode + bandwidth on tune

- Mode: set from `station.mode` (AM for most, USB for CHU).
- Bandwidth: 5 kHz for AM stations, 2.4 kHz for USB.  Reuses
  the operator's stored RX BW for that mode rather than
  hardcoding (operator preference wins).
- Volume / AGC unchanged — operator's current settings carry over.

### 2.5 Settings → Bands → Time Stations

Per-operator preferences:
- Reorderable favorites list (drag to rearrange)
- Per-station hide toggle (won't appear in cycle)
- Auto-AM-bandwidth override (5 kHz default, range 3-10 kHz)

## 3. GEN1 / GEN2 / GEN3 customizable buttons

### 3.1 Current behavior

GEN1 / GEN2 / GEN3 currently jump to hardcoded frequencies
defined in `lyra/bands.py`.

### 3.2 New behavior

Each GEN button gets a per-button QSettings entry:

```
QSettings keys:
  bands/gen1_freq_hz  (int, default 4625000)
  bands/gen1_mode     (str, default "USB")
  bands/gen1_label    (str, optional 12-char display label)
  bands/gen2_freq_hz / gen2_mode / gen2_label
  bands/gen3_freq_hz / gen3_mode / gen3_label
```

**Click behavior:**

- **Plain left-click**: tune to saved freq + mode.
- **Right-click**: popup menu:
  - **"Save current freq + mode here"** — opens confirm dialog
    showing what will be saved + asks for an optional label
  - **"Set custom label..."** — text input dialog
  - **"Reset to default"** — restore the hardcoded value
  - **"Cancel"** — close menu

### 3.3 Confirm dialog (save flow)

```
┌─ Save GEN1 preset? ──────────────────────┐
│                                          │
│  Save current frequency + mode to GEN1?  │
│                                          │
│    Current: 7.250.000 MHz, LSB, 2.4 kHz │
│                                          │
│    Existing: 4.625.000 MHz, USB         │
│                                          │
│  Optional label (12 chars max):          │
│    ┌──────────────────────────────────┐  │
│    │                                  │  │
│    └──────────────────────────────────┘  │
│                                          │
│             [ Cancel ]  [ Save ]         │
└──────────────────────────────────────────┘
```

Cancel = no-op.  Save = overwrite QSettings entry, button label

### 3.4 Tooltip

Hovering a GEN button shows the saved freq, mode, and label:
```

```

## 4. User memory presets (20 slots)

### 4.1 UX — single Presets button on Bands panel

New button on the Bands panel: **`Presets ▾`** (with dropdown
caret).  Sits next to the GEN1 / GEN2 / GEN3 cluster.

**Plain click:** opens the dropdown menu.  Lists all saved
presets (up to 20) by name.

**Click any item:** tunes to that preset's freq + mode.

**Right-click:** opens an extended menu:
- **"Save current as new preset..."** (greyed out when at 20 max)
- **"Manage presets..."** — opens Settings → Bands → Memory tab
- **"Sort presets..."** — submenu: by name / by freq / by mode
- Separator + the same preset list (so right-click always lets
  you tune AND access management in one menu)

### 4.2 Save flow

"Save current as new preset..." opens a dialog:

```
┌─ Save preset ────────────────────────────┐
│                                          │
│  Saving:  7.250.000 MHz, LSB, 2.4 kHz   │
│                                          │
│  Name (required, 30 chars max):          │
│    ┌──────────────────────────────────┐  │
│    │                                  │  │
│    └──────────────────────────────────┘  │
│                                          │
│  Notes (optional, 80 chars max):         │
│    ┌──────────────────────────────────┐  │
│    │                                  │  │
│    └──────────────────────────────────┘  │
│                                          │
│             [ Cancel ]  [ Save ]         │
└──────────────────────────────────────────┘
```

Empty name = save button greyed.  Name collision = prompt to
overwrite or rename.

### 4.3 Storage schema

```
QSettings key: bands/user_presets
Value: JSON-serialized list of dicts:

[
  {
    "name": "MY 40M FT8",
    "notes": "weekend digital ops",
    "freq_hz": 7074000,
    "mode": "DIGU",
    "rx_bw_hz": 3000  // optional; null = use mode default
  },
  ...
]
```

**Constraints:**
- Max 20 entries — beyond that the "Save current as new preset"
  is greyed in the menu.  Operator must delete an entry first.
- Names must be unique (case-insensitive); save dialog warns
  on collision.
- 30-char name max, 80-char notes max — keeps the dropdown
  readable.

### 4.4 Settings → Bands → Memory tab

Full management UI for the 20 presets:

- Table view: # | Name | Freq | Mode | BW | Notes
- Buttons: Edit | Delete | Duplicate | Move Up | Move Down
- Import / Export to CSV (operator can edit in Excel)
- Order in this list IS the order in the Presets dropdown

### 4.5 Import / Export

CSV format (header row + data):
```
Name,Freq_Hz,Mode,RX_BW_Hz,Notes
"MY 40M FT8",7074000,DIGU,3000,"weekend digital ops"
"WWV 10 MHz",10000000,AM,5000,""
```

Import path:
1. Operator clicks Import in Settings.
2. File picker opens.
3. Parser validates each row.  Bad rows reported with line number,
   skipped.  Valid rows merged with existing presets (duplicates
   skipped or prompt-overwrite per-row).
4. If total exceeds 20, prompt operator to choose which to keep.

Export path: trivial — dump current presets list to CSV with
file picker for destination.

## 5. SW broadcaster overlay (EiBi)

The big one in this release.  Substantial work but architecturally
clean.

### 5.1 EiBi format

Free, monthly-updated CSV from <https://www.eibispace.de/>.  Plain
semicolon-delimited.  Header row:

```
KHZ;TIM;DAY;ITU;STN;LANG;TGT;REM;P;START;STOP
```

| Field | Meaning |
|---|---|
| `KHZ` | Frequency in kHz (integer) |
| `TIM` | Time-on-air UTC, e.g. "1500-1600" or "0000-2400" |
| `DAY` | Day of week, "1234567" pattern, blank = daily |
| `ITU` | ITU country code (e.g. "USA", "GBR", "RUS") |
| `STN` | Station name (e.g. "Voice of America") |
| `LANG` | Language code (e.g. "E"=English, "S"=Spanish) |
| `TGT` | Target area (e.g. "EaAs", "Eu") |
| `REM` | Site/transmitter remarks |
| `P` | Power class (0=<50kW, 1=50-100kW, 2=100-250kW, 3=250+kW) |
| `START` / `STOP` | YYMMDD seasonal validity |

~30,000 entries per file.  Files named by season:
`sked-A26.csv` (April-October 2026), `sked-B26.csv`
(October 2026 - March 2027).

**License:** free for non-commercial use, attribution required
("Source: EiBi <https://www.eibispace.de/>").  Compatible with
GPL distribution as long as we DON'T bundle it (operator
downloads themselves).  Same model SDR# / SDRuno follow.  No
licensing risk.

### 5.2 Architecture

```
lyra/swdb/                                 ← new package
├── __init__.py
├── eibi_parser.py        ← CSV → list[EibiEntry]
├── store.py              ← in-memory index + on-disk persistence
├── downloader.py         ← https GET with progress + checksum
└── time_filter.py        ← "is on air at this UTC time?" predicate

lyra/ui/spectrum.py + spectrum_gpu.py
├── extend draw loop to render station labels
└── extend hover tooltip with station details

lyra/ui/settings_dialog.py
└── New "SW Database" tab in the Bands category
```

### 5.3 EibiEntry dataclass

```python
@dataclass(frozen=True)
class EibiEntry:
    freq_khz: int
    time_start: int    # minutes UTC since 00:00 (e.g. 0930 = 570)
    time_stop: int
    days: frozenset[int]  # 1..7, empty = daily
    itu_code: str
    station: str
    language: str
    target: str
    remarks: str
    power_class: int   # 0..3
    valid_from: date
    valid_to: date
```

### 5.4 In-memory index

Sorted-by-frequency list with binary search by visible panadapter
range.  ~30k entries × ~200 bytes/entry = ~6 MB resident.

```python
class EibiStore:
    def load(self, path: Path) -> int: ...   # returns row count
    def lookup_in_range(
        self, freq_lo_khz: int, freq_hi_khz: int,
        utc_now: datetime,
        min_power: int = 1,
    ) -> list[EibiEntry]: ...
    @property
    def last_updated(self) -> Optional[datetime]: ...
    @property
    def source(self) -> str: ...   # 'EiBi A26' etc.
```

Per-frame lookup cost: O(log N + K_visible).  At a 24 kHz
panadapter span on 40m, K ≈ 30-50 entries.  Trivially fast.

### 5.5 Downloader

```python
class EibiDownloader(QObject):
    progress = Signal(int, int)   # bytes_done, bytes_total
    finished_ok = Signal(Path)    # downloaded file path
    finished_error = Signal(str)

    URL_PATTERN = "https://www.eibispace.de/dx/sked-{season}.csv"

    def fetch(self, season: str = "auto") -> None:
        """Background HTTPS GET with 30 sec timeout.  'auto' picks
        the season that covers today's date."""
```

Storage location: `QStandardPaths.AppLocalDataLocation / "swdb"`.
On Windows: `%APPDATA%\Lyra-SDR\swdb\sked-A26.csv`.

### 5.6 Panadapter overlay

Reuse the existing TCI spots renderer for collision-free row
packing.  Same patterns:

- Render labels at station's freq position within visible range
- Stack rows when multiple stations close together
- Hover tooltip shows full station info (station / lang / target /
  power / time-on-air)
- Click-to-tune already wired (existing landmark click path)

**Rendering filters:**
- Time-of-day filter: only render stations currently on-air per
  their `time_start`-`time_stop` window in the current UTC time
- Power filter: default `min_power >= 1` (50+ kW), operator-tunable
  in Settings (0 = show everything, including small relays)
- Language filter: optional whitelist (e.g. "show only English/Spanish")
- Per-band toggle: operator can disable overlay on bands where
  it's noisy clutter (e.g., 40m crowded with broadcasters)

**Default visibility:** OFF on first install — operator opts in
via Settings → Bands → SW Database → "Enable overlay."  Reason:
clutter on bands the operator doesn't care about; opt-in keeps
the default UX clean.

### 5.8 Update frequency

EiBi seasons run April-October (A) and October-March (B).
Recommended download cadence: monthly.  The Settings tab shows
"X days old" so the operator can see when to refresh; we don't
auto-download (network-policy and bandwidth respect — operator
opts in each time).

## 9. Risk register

| Risk | Severity | Mitigation |
|---|---|---|
| EiBi download fails (firewall, network) | Low | Settings UI shows error, lets operator manually drop CSV in `%APPDATA%\Lyra-SDR\swdb\` |
| Overlay clutters busy bands | Med | Per-band disable toggles + power-class filter + time-of-day filter (all default-on) |
| 30k entries × every panadapter frame = perf hit | Low | Sorted-by-freq index + binary search; bench during step 4 |
| EiBi license change | Low | Free / non-commercial / attribution model has held since 2002; we don't redistribute the file |
| 20-preset limit feels small | Low | Trivial to bump later; observe operator usage first |
| Confirm dialog too friction-heavy on GEN save | Low | Operator explicitly requested confirm; can revisit if it bugs them |
